# 文章阅读


[**Qcon北京2018《容器生态下的高性能负载均衡建设之路》-韩建飞.pdf**](https://ppt.geekbang.org/list/qconbj2018)
* JDOS 京东数据中心操作系统
* ContainerLB研发背景
   * 性能：业务日益增长的高并发、低延迟需求
   * 成本：同样流量下，需要更少的负载均衡器，节约成本
* ContainerLB目标：
   * 接近网卡线速的报文转发性能
   * 支持常见的4层负载均衡功能，配置灵活
   * 充分发挥双路服务器性能，支持两个zone的网卡避免计算资源闲置
   * 融入JDOS生态，落地K8S Service
   * 部署简单，避免改动客户端和后端服务，业务应用无感知
   * 负载均衡集群高可用，做到条件热升级
* ContainerLB功能特性
   * 协议：负载均衡支持TCP/UDP
   * 链路跟踪：支持TCP链路跟踪，记录链路状态，支持快速会话释放
   * 高可用：支持容器的健康检查，保证应用可用性
   * 集群部署，多层次容错机制
   * 灵活性：多种流量调度算法
   * 易用性：多种管理途径，轻松操纵负载均衡
* 调度算法
   * source-ip：源ip hash调度
   * source-ip-port：源ip+port一致性hash调度
   * least-connection：最小连接数调度
   * round-robin：轮询调度
   * weighted round-robin：加权轮询调度
* ContainerLB集群架构
   * 支持DPDK LB
   * 支持Haproxy/Nginx
   * 支持融入kubernetes的集权管理
   * 支持单独的DPDK LB Driver驱动单独集群
   * Haproxy/Nginx负载均衡容器化自动扩容和迁移
* 性能提升
   * DPDK本身的性能优势，poll mode，无锁队列，内存零拷贝，用户态驱动
   * 尽量少的避免锁的使用
   * 恰当地使用rte_prefetch0()，可以减少cache-miss次数，避免当前函数成为性能热点
   * 恰当地使用likely()和unlikely()，可以减少分支预测失败的次数


[**QCon北京2018-《如何开发基于AI的聊天机器人应用》-黄冠辉**](https://ppt.geekbang.org/list/qconbj2018)
* 背景介绍
   * 消息通信是最常用，而且最熟悉的环境
   * Gartner 2017：机器人讲开始取代移动应用程序。用户将不再寻找应用程序、下载应用程序、更新或管理应用程序
   * 聊天机器人市场估值从2016年的7亿美金，发展到2021年的31.7亿
* 常规任务自动化，让人类处理复杂任务
   * 聊天机器人可以处理数百万计的常规问题，但无法处理复杂逻辑
   * 复杂逻辑交给员工专心处理
* 聊天机器人典型对话过程
   * 用户输入
   * 用户意图？
   * 提取关键参数
   * 对话流程，执行、响应
* 意图系统
   * 用户想要什么，下面这些，都要执行“订票”动作
      * 我要订去上海的机票
      * 明天飞深圳的机票有哪些
      * 帮我看看去成都的机票
      * 我要飞上海
   * 这可以如何分类一个动作
   * 训练意图模型，运行时检测用户意图
* 实体系统
   * 解析句子
   * 取出相关信息
   * 实体：出发地，目的地，起飞日期
   * 实体模型
      * 基于话语训练数据
      * 实体系统分析并提取一个句子中的重要信息
* 对话流程
   * 管理对话流程
   * 存储会话状态和数据上下文
   * 调用自定义业务逻辑
* 甲骨文聊天机器人平台的关键组件
   * 渠道：聊天机器人与人之间的统一跨渠道聊天界面
   * 人工智能引擎：NLP和ML，语义识别和实体提取
   * 业务系统集成：SDK调用后端API，并提供来自聊天应用程序用户的问题的答案
   * 对话流程执行：声明式聊天工具构建用户界面，执行工作流的状态机
* 聊天机器人最佳实践
   * 使用丰富的消息格式、表单、对会话进行补充
   * 记住会话的上下文
   * 从最开始通过分析来监控您的聊天机器人
   * 识别用户常用，能形成习惯的场景
   * 不要让用户直接训练人工智能
   * 转接人工客服的选项


# 业界特快

[**Facebook 的专利**](https://github.com/ruanyf/weekly/blob/master/docs/issue-41.md)
* 新闻媒体发现，Facebook 为了找出人与人之间的关系，已经申请了一系列令人意想不到的专利
* 一个专利是
   * Facebook 如果检测到两部智能手机在同一个时间同一个地方出现，通过比较每部手机的加速度计和陀螺仪读数，可以确定两个人是面对面还是一起散步
   * 这样的话，Facebook 可能就会推荐给你昨晚在酒吧与你交谈的朋友，而不是没有交谈过的人
* 另一个专利是
   * 通过用户上传的照片元数据连接两个人
   * 如果不同用户上传的图像，看起来像是同一系列的照片，比如文件名是IMG_4605739.jpg 和 IMG_4605742.jpg
   * 或者如果在照片上的相同位置可以检测到镜头划痕或灰尘，那么可能会假设两个人彼此认识，因为他们使用了同一台相机拍照
